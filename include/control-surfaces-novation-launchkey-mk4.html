General behavior

The control surface comes with a number of predefined mappings that are typically done either with a MIDI map or with MIDI Learn:

  • Track selection: you can press left-pointing and right-pointing buttons to select the previous or the next track.
  • Basic transport controls: rolling, stopping, toggling rec-mode.
  • Workflow buttons: Undo/Redo (Shift+Undo), Quantize, Metronome toggle. Note: Capture MIDI is not a function available in Ardour, thus unsupported.
  • Faders: 8 faders are mapped to the first 8 tracks, 1 fader is mapped either to the monitor (if the monitor section exists) or the master bus (if there is no monitor section).

Modes support

Plugin Encoders control plugin parameters, right now i don't recall how to navigate between plugins
Mixer Encoders can be used for pan, the first 2 sends, and additional gain controls; pads do solo & mute
Sends Encoders control 8 sends of the selected track
Transport Encoders control playhead, loop points, can be used to jump between markers, visual zoom
DAW Pads function to control 2 scenes of 8 tracks (can "scroll around" to get to others), including scene launch and individual cue/clip/slot launching

Please note that additional modes — Drum, User Chord, Chord Map — do not require DAW involvement. All MIDI processing happens inside the box, the keyboard only sends final MIDI events.
